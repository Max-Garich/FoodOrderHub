# FoodOrderHub 🍽️

Система заказа еды с управлением балансом.

## Особенности
- **Мобильно-ориентированный дизайн**: Современный интерфейс с эффектами Glassmorphism.
- **Безопасность**: Баланс контролируется только администратором. Никаких платежных систем — всё на доверии и ручном пополнении.
- **Отчетность**: Автоматическая генерация отчетов по окончании сессии заказов.
- **Справочник блюд**: Удобное управление меню на каждый день.

## Технологический стек
- **Frontend**: React, Vite, Vanilla CSS.
- **Backend**: Node.js, Express, Prisma.
- **Database**: PostgreSQL.
- **Infrastructure**: Docker & Docker Compose.

## Быстрый старт

### Требования
- Docker
- Docker Compose

### Запуск
1. Склонируйте репозиторий.
2. Создайте файл `.env` на основе `.env.example`.
3. Запустите проект:
   ```bash
   docker-compose up -d --build
   ```
4. Приложение будет доступно по адресу: [http://localhost:3000](http://localhost:3000)

## Локальная разработка и Ngrok 🌐

Если вам нужно предоставить доступ к проекту через интернет (например, для тестирования на мобильном телефоне), вы можете использовать встроенную интеграцию с **ngrok**.

### Настройка
1. Установите зависимости в корне проекта:
   ```bash
   npm install
   ```
2. Добавьте ваш `NGROK_AUTHTOKEN` в файл `.env` (получить можно на [dashboard.ngrok.com](https://dashboard.ngrok.com/)).

### Запуск
*   **Только туннель**:
    ```bash
    npm run tunnel
    ```
*   **Сервер + Туннель (одновременно)**:
    ```bash
    npm run dev:all
    ```
    *Примечание: Перед этим убедитесь, что база данных запущена (например, через docker-compose).*

## Лимиты и правила
- Заказ невозможен, если сумма превышает текущий баланс пользователя.
- Заказ невозможен, если администратор закрыл сессию приёма заказов.
- Баланс пользователя не может быть изменен самим пользователем.
